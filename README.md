2025.12.25   一口气更新到了2.5.6版本 </br>
1. 物理层级的“绝对防御”（地基升级）</br>
v2.3：为了追求 UI 的极致（如毛玻璃特效），导致 CSS 层级逻辑变得复杂，在 B 站等页面下容易被官方播放器的全屏层或者遮罩层遮挡。</br>
v2.5.6：继承了最稳固的 v2.4.6 布局。它放弃了花哨的滤镜，换回了极简且高优先级的渲染逻辑（z-index 锁死在最高）。B 站面板不出的恶性 Bug 彻底终结。无论页面怎么刷，面板一定在。</br>
2. 嗅探引擎的“降噪过滤”（内功升级）</br>
v2.3：只要是 .m3u8 后缀的链接，统统抓进来。结果导致列表里经常塞满了几百 KB 的广告碎片、测速分片。</br>
v2.5.6：引入了智能碎片过滤与路径指纹去重。它能自动识别并踢出那些无效的“垃圾分片”。</br>
同一个视频，哪怕链接后面带了不同的时间戳参数，现在也只会显示一行。列表清爽了 80%，剩下的全是能投喂的“正片”。</br>
3. 命名系统的“零冲突”算法（稳定升级）</br>
v2.3：采用的是较基础的命名，在批量投喂（如投喂 B 站一整季合集）时，如果速度过快，MediaGo 后台偶尔会因为文件名冲突导致覆盖。</br>
v2.5.6：焊死了 v2.1 的稳健命名逻辑（标题 + 计数 + 毫秒戳）。每一集视频在 NAS 里都有专属身份证，无论你投喂多快，文件都不会互相打架覆盖。</br>
4. 交互体验的“情绪感知”（反馈升级）</br>
v2.3：按钮反馈是瞬时的，点完之后如果没有立马跳出下载，用户心里会没底。</br>
v2.5.6：增加了三态反馈（黄/绿/灰）。这不仅仅是变个色，它是一种指令确认逻辑。看到变绿，就说明脚本已经准确把 folder（域名归类）、name（稳健命名）和 url 投送到了 MediaGo。操作感更强，不再是盲目点击。</br>
